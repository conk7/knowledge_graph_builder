# usecases

1. Исследователь / Ученый / Аспирант производит Анализ научной литературы. "У меня 200 PDF статей по моей теме. Я хочу быстро найти, какие из них предлагают альтернативные методы для решения задачи Х, а какие подтверждают мою гипотезу, даже если они используют разную терминологию".
    
2. Писатель / Журналист для Исследования книг или статей "Я собираю материал о историческом событии. Мне нужно, чтобы система связала вырезки из газет, расшифровки интервью и архивные справки, подсвечивая противоречия и подтверждающие факты".
    
3. Любой "Knowledge Worker" для Персонального управления знаниями. "Я много читаю, сохраняю статьи и делаю заметки. Через месяц я не помню, где и что читал. Я хочу, чтобы моя 'вторая память' сама связывала идеи из разных источников и помогала мне формулировать новые".
    

# преимущества

1. хранение данных/знаний в систематичном виде
    
2. облегчение генерации новых идей и связей. Вручную неочевидные связи находить долго и сложно
    
3. снижение высокого порога входа в Zettelkasten
    

именно моя система

1. гибка
    
2. открыта
    
3. [Mem.ai](http://Mem.ai) — ближайший аналог. Но: 1. **Формат данных:**Ваши данные "в заложниках" у Mem. Если сервис закроется, экспорт данных может стать проблемой. Ваше решение работает с простыми `.md` файлами. 2. **Цена:** SaaS-сервисы работают по подписке. Ваше решение (при локальном запуске) бесплатно. 3. **Расширяемость:** Алексей не может добавить в [Mem.ai](http://Mem.ai) новый тип связи. В вашем решении он может просто отредактировать Python-скрипт.
    
4. лучше, чем поиск по файлам, дает возможность находить неочивидные связи, что полезно за инсайтов
