Вам дан массив целых чисел `coins`, представляющий монеты разных номиналов, и целое число `amount`, представляющее общую сумму денег.  
Напишите функцию для вычисления **минимального** количества монет, необходимых для составления этой суммы. Если сумму нельзя составить ни одной комбинацией монет, верните -1.  
Предполагается, что у вас бесконечное количество монет каждого номинала.

**Пример:**  
Вход: `coins = [1, 2, 5]`, `amount = 11`  
Выход: `3` (5 + 5 + 1)

Решение:
Используем подход "снизу вверх" (bottom-up) динамического программирования. Мы создаем массив `dp`, где `dp[i]` хранит минимальное количество монет для суммы `i`.  
Базовый случай: для суммы 0 нужно 0 монет. Для любой другой суммы `i` мы проверяем каждую монету `c`: `dp[i] = min(dp[i], dp[i - c] + 1)`.

```python
def coinChange(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    
    for i in range(1, amount + 1):
        for coin in coins:
            if i - coin >= 0:
                dp[i] = min(dp[i], dp[i - coin] + 1)
                
    return dp[amount] if dp[amount] != float('inf') else -1
```

**Разбор:**

- Сложность по времени: `O(S * n)`, где `S` — сумма (amount), а `n` — количество номиналов монет.
- Сложность по памяти: `O(S)` для массива `dp`.
